L’écosystème front-end est en permanente évolution, depuis plusieurs années un débat est ouvert entre différentes façons de créer des projets/site web. On nous a parlé de SSR, SPA, MPA, Web Component… 

Certains framework comme React propulsait le Client Side Rendering et la mécanique de SPA, puis depuis un virage à 180 degrés et sa version 18, préconise maintenant le Server Side Rendering afin d’améliorer sa Web Performance. 
Mais qu’elle est donc la différence entre tous ces termes barbares que sont le SSR, CSR et Hybrid Rendering ? Comment chacune de ses solutions de rendu peut avoir un impact sur la Web Performance? 
Dans quel cas est-il préférable d’utiliser une solution de rendu plutôt qu’une autre?

Vous vous posez certainement cette question, comment allier le meilleur de tous ces mondes ?

Vous verrez comment nous avons traité cette question de notre côté, quel framework/technologies ont été choisi pour répondre à quel usage tout en respectant la Web Performance! 
Lors de cette conférence, vous découvrirez aussi comment et pourquoi nous avons crée notre propre solution pour concevoir et développer des micro front-end permettant de faire de l’Hybrid Rendering à l’aide de NestJS et Svelte tout en conservant la performance d’une technologie robuste comme le JAVA en SSR.

Et parce que la performance est pour tous, nous vous montrerons aussi comment nous avons à l'aide de Vite améliorer l'expérience de développement de nos utilisateurs.
